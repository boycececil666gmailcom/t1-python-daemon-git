## Call Flow

```
1. cli.py — main() → cli()   Click group dispatches on argv subcommand
   │
   ├─► scan subcommand
   │     └─ 2. scanner.scan_drives(drives, progress_callback)
   │              └─ 3. _available_drives()          ctypes GetLogicalDrives bitmask → ["C:\\", "D:\\", …]
   │              └─ 4. os.walk() per drive           skips system/hidden dirs
   │                        └─ 5. _is_git_repo()      checks for .git/ folder (blocking FS check)
   │                        └─ 6. _remote_url()       git remote get-url origin (subprocess, blocking)
   │              returns: [{"path", "remote"}, …]
   │     └─ 7. registry.register(path, remote)        upserts to ~/.daemon-git/registry.json
   │     └─ 8. _run_sync_all()                        (unless --no-sync)
   │
   ├─► sync subcommand
   │     └─ 9. _run_sync_all(verbose)
   │              └─ 10. registry.all_repos()          reads registry.json → list[dict]
   │              └─ 11. syncer.pull_repo(path)         per repo (blocking, ≤120 s)
   │                        └─ 12. _has_remote()        git remote (subprocess)
   │                        └─ 13. _remote_url_for_pull()   git remote get-url origin
   │                        └─ 14. auth.git_auth_args(remote_url)
   │                                   └─ host_from_remote()  regex/urlparse
   │                                   └─ get_account(host)   reads accounts.json
   │                                   returns ["-c", "http.extraheader=Authorization: Basic …"]
   │                        └─ 15. git pull --ff-only  with injected auth args (subprocess)
   │                        └─ 16. _has_conflict_markers()   string-scan stdout+stderr
   │                        returns: SyncResult(path, SyncStatus, message)
   │              └─ 17. registry.update_status()      writes last_synced + last_status
   │
   ├─► daemon subcommand
   │     └─ infinite loop: sleep(interval) → _run_sync_all()
   │              optionally: scanner.scan_drives() every N cycles (--rescan-every)
   │
   ├─► auth add / remove / list
   │     └─ auth.add_account / remove_account / list_accounts
   │              └─ reads/writes ~/.daemon-git/accounts.json
   │              └─ _restrict_permissions() via icacls (Windows owner-only perms)
   │
   └─► add / remove / list   direct registry reads/writes
```

## Key APIs

| Name | File | What it does | Why critical |
|---|---|---|---|
| `scan_drives()` | `daemon_git/scanner.py` | Walks all drive roots with `os.walk`, prunes system dirs, detects `.git/`, returns paths + remote URLs | Only discovery mechanism — everything else depends on its output |
| `registry.register()` / `all_repos()` | `daemon_git/registry.py` | Idempotent upsert to `registry.json`; `all_repos()` is sole source of truth for what to sync | Every sync reads from this; losing it means the daemon forgets all repos |
| `syncer.pull_repo()` | `daemon_git/syncer.py` | Runs `git pull --ff-only` with auth injected; detects conflicts via string matching → returns typed `SyncResult`; **never overwrites on conflict** | Core safety contract: conflict detection prevents data loss |
| `auth.git_auth_args()` | `daemon_git/auth.py` | Maps remote URL host → stored credentials → `-c http.extraheader=Basic …` args for git | Without it, pulls against private HTTPS remotes fail silently |
| `_run_sync_all()` | `daemon_git/cli.py` | Orchestrates full sync loop: load registry → pull each repo → update status → print summary | Used by both `sync` and `daemon` commands |
